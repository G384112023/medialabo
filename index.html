「Webコンテンツ作成」の成果

本テーマについて

目的は、先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得することである。

学習目標は、

  • HTML によって Web ページの構造を記述する基本的な方法がわかる
  • CSS によって Web ページの見た目を設定する基本的な方法がわかる
  • JavaScript によって Web ページのインタラクティブな処理を記述する基本的な方法がわかる
  • HTML, CSS, JavaScript の構文リファレンスから，構文の書式や使用方法の情報を入手できる
  • HTML, CSS, JavaScript, 画像・音声・動画ファイルを組み合わせて簡単な Web ページが作成できる
である。

プロジェクト課題について

プロジェクト課題で作成したページ: project.html

都市一覧から1つ都市を選び、テキストボックスに都市名を入力しserchボタンを押すと、その都市の天気が表示される。 表示される項目は経度、緯度、天気、最低気温、最高気温、湿度、風速、風向、都市名である。その右側に天気に合ったイラストを表示する。 都市一覧以外のテキストを入力したり、何も入力せずに検索すると、検索できないというメッセージが表示される。 検索は日本語でも英語でも可能であり、イラストも表示されるため、海外の人でも利用可能である。

数当てゲーム

数当てゲームのページ: kazuate.html

数当てゲームは、プレイヤーが1から10までの整数の中から1つの数字を選び、テキストボックスにその数字を入力して、ランダムに選ばれた答えの数字と一致しているかを当てるゲームである。 プレイヤーが数字を入力して送信すると、その数字を判定し、正解か不正解かの結果が表示される。 ゲームは最大3回までチャレンジすることができ、1回目と2回目で不正解だった場合には、「答えの数字は大きい」または「小さい」といったヒントが表示され、3回目も不正解だった場合には、正解の数字が表示され、ゲームは終了する。 1回目から3回目の間に正解することができれば、ゲームは成功となり、「正解です. おめでとう！」と祝福メッセージが表示される。

各授業回の学習事項

第1回 WWW と HTML

今回の講義ではWWWとHTMLについて学んだ。Webページを提供する仕組みのことWWWという。Webページの構造を記述するための書式をHTMLといい、HTML 文書を作ることによって，Webページに掲載するコンテンツと，ページの基本的なレイアウトを設定できる。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

今回の講義では、CSSを用いたWebページの装飾方法について学んだ。CSSを使うことで、文字や背景の色、フォントの大きさ、要素の幅や高さ、枠線のスタイルなどを自由に設定することができ、ページ全体の見た目を整えることが可能となる。また、HTMLのhead要素内で外部のCSSファイルを指定することで、デザインを適用することができる。

演習の成果: 演習2-3: Webブラウザ説明ページの装飾

第3回 JavaScript の基本

今回の講義ではjavascriptについて学んだ。JavaScript は Web ページ上で複雑な機能を実現するプログラミング言語である。 Web ページを開いた直後や，ボタンを押したときに JavaScript プログラムが実行される。ページの自動更新やアニメーションなど動きのあるページを実現するために JavaScript プログラムを使う。

演習の成果: 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

今回の講義ではフォーム要素とイベント処理とDOMについて学んだ。ページ上に設置するテキストボックスとボタンをHTMLで記述し、ボタンを押した後の処理をjavascriptで記述する。 また、ページに設置されているHTML要素の検索とテキスト変更をDOMを使用することで実現できる。

演習の成果: 演習4-10: 足し算ページ

第5回 イベント処理

今回の講義ではDOMの高度な機能について学んだ。要素の新規作成、要素の編集、要素の削除をDOMを用いて行うことができる。これによって、javascriptを通じてページを自由に変更することができるようになる。

演習の成果: 演習5-7: DOMプログラミング2

工夫点・苦心点・強調点・感想

今回の制作では、思い描いた機能を実現するために多くの試行錯誤を重ねた。特に苦労したのは、テキストボックスに入力された内容の条件分岐である。nullを使って処理を分けようと試みたものの、なかなか思ったように動作せず、原因の特定に時間を要した。入力が空かどうか、あるいは入力された都市名がデータに存在するかどうかを適切に判断するために、nullを使って判定を行ったが、うまく動作せず、プログラムを見直すことになった。
また、ページのヘッダー部分のデザインについても迷いがあった。当初はh1タグを使ってタイトルを表示する形でも問題ないと考えていたが、ページ全体の見た目や情報の整理を意識した結果、header要素を使ってレイアウトを整えることにした。headerを使うことで、より構造的にもデザイン的にも整ったページに仕上げることができた。
画像の挿入にも大きな壁があった。検索結果に対応するイラストを表示したかったが、どのようにして画像を切り替えるかが分からず、かなりの時間を費やした。最終的には、画像のsrc属性にリンクを直接指定することで、条件に応じた画像を表示できるようになった。これはidなどで値を操作する方法と同じように考えることで、ようやく解決できた課題である。
検索結果のレイアウトにも工夫を凝らした。具体的には、テキストを左側に、対応するイラストを右側に表示する段組みの構成とし、それぞれがバランスよく見えるようにmarginやpadding、widthなどを細かく調整した。また、イラストのサイズが天気の種類によって異なっていたため、widthとheightを個別に設定し、見た目が揃うように工夫した。これらの細かな調整は想像以上に時間がかかってしまった。
そのほか、検索ボックスに何も入力されていない場合や、登録されていない都市名が入力された場合には、「検索できませんでした」と明確に表示させるようにした。また、天気に対応するイラストを大きく表示することで、直感的に天気の情報が伝わるようにした。このイラストはPowerPointの図形機能を使って自作したものである。
さらに、日本語と英語の両方で都市名を検索できるように対応させたことにより、日本人だけでなく、英語を使うユーザーにも利用しやすいものとなった。検索ボタンのテキストを英語で「search」としたのも、そういった利用者を意識した工夫である。また、都市一覧や検索結果を枠線で囲むことで、それぞれの情報がまとまりとして認識しやすくなるように配慮した。
全体として、実現したい機能を形にするには、HTML、CSS、JavaScriptの3つの言語を同時に意識しながら開発を進める必要があり、コードの整合性を保つのが大変だった。ちょっとした変更が予期しない不具合を引き起こすことも多く、そのたびに原因を探して修正する作業が発生した。また、講義資料や参考資料を参照しながらの作業だったため、想定よりもかなり時間がかかってしまった。だが、そうした困難を乗り越えて、狙い通りに機能が動いたときの達成感はかなり大きかった。
